# LLM Wiki Schema

本项目采用 LLM Wiki 方法论构建个人知识库。

## 核心原则

**LLM 维护，人类主导。** 我负责知识库的日常维护：创建页面、更新索引、交叉引用、标注矛盾。你负责：提供信息源、提出问题、探索关联、思考意义。

## 三层架构

```
/Users/zhangqingzheng/code/llmwiki/aiwiki/
├── raw/                    # 原始素材（只读）
│   ├── articles/           # 文章、论文
│   ├── books/              # 书籍章节
│   ├── images/             # 图片资产
│   └── data/               # 数据文件
├── wiki/                   # 知识库主体（LLM 维护）
│   ├── index.md            # 内容索引
│   ├── log.md              # 操作日志
│   ├── overview.md         # 知识库概览
│   ├── entities/           # 实体页面（人物、组织、概念）
│   ├── topics/             # 主题页面（按领域组织）
│   ├── sources/            # 来源摘要（每篇原文一个页面）
│   └── synthesis/          # 综合分析（跨来源的观点、对比）
└── AGENTS.md               # 本文件（LLM 行为规范）
```

## 工作流程

### Ingest（摄入）

当你添加新素材到 `raw/` 目录时，我会：

1. 读取素材，提取关键信息
2. 与你讨论要点，确认理解方向
3. 创建来源摘要页面（`wiki/sources/`）
4. 更新相关实体页面、主题页面
5. 检查与现有知识的矛盾或补充
6. 更新 `index.md` 和 `log.md`
7. 一次摄入可能影响 10+ 个 wiki 页面

### Query（查询）

当你提问时，我会：

1. 先读 `index.md` 定位相关页面
2. 深入阅读相关内容
3. 综合分析并给出带引用的答案
4. 有价值的分析会被写入 wiki（查询结果也沉淀为知识）

### Lint（健康检查）

定期检查 wiki 状态：

- 页面间的矛盾
- 过时的主张（被新来源取代）
- 孤立页面（无入链）
- 缺失的概念页面
- 缺失的交叉引用
- 可填补的数据空白

## 页面格式约定

### 来源摘要（`wiki/sources/`）

```markdown
# [来源标题]

**类型**: article/book/paper/video
**日期**: YYYY-MM-DD
**来源**: URL 或出处
**标签**: #tag1 #tag2

## 摘要
[核心观点，200-400字]

## 关键要点
- 要点 1
- 要点 2
- 要点 3

## 关联实体
- [[实体A]] - 关系说明
- [[实体B]] - 关系说明

## 引用此来源的页面
- [[主题页面X]]
- [[实体页面Y]]
```

### 实体页面（`wiki/entities/`）

```markdown
# [实体名称]

**类型**: person/org/concept/tool
**创建日期**: YYYY-MM-DD
**最后更新**: YYYY-MM-DD
**来源数量**: N

## 定义
[简短定义，50-100字]

## 核心观点/特征
- 特征 1
- 特征 2

## 与其他实体的关系
- [[实体A]] → 关系
- [[实体B]] ← 关系

## 相关来源
- [[来源X]] - 要点摘录
- [[来源Y]] - 要点摘录

## 讨论与争议
[如有]

## 开放问题
- 问题 1
- 问题 2
```

### 主题页面（`wiki/topics/`）

```markdown
# [主题名称]

**创建日期**: YYYY-MM-DD
**最后更新**: YYYY-MM-DD
**来源数量**: N

## 概述
[主题综述，200-400字]

## 核心概念
- [[概念A]]
- [[概念B]]

## 主要观点演变
[按时间或逻辑顺序]

## 相关来源
- [[来源X]]
- [[来源Y]]

## 待探索方向
- 方向 1
- 方向 2
```

## 索引文件（`index.md`）

按类别组织所有页面，每个条目包含：链接、一句话摘要、可选元数据。

```
## 来源摘要 (N)
- [[sources/article-1|文章标题]] - 一句话摘要
- [[sources/paper-2|论文标题]] - 一句话摘要

## 实体页面 (N)
### 人物
- [[entities/person-a|人物A]] - 简介
### 概念
- [[entities/concept-x|概念X]] - 定义

## 主题页面 (N)
- [[topics/topic-1|主题1]] - 描述
```

## 日志文件（`log.md`）

按时间顺序记录操作：

```markdown
## [YYYY-MM-DD HH:MM] ingest | 文章标题
- 创建 [[sources/article-x]]
- 更新 [[entities/person-a]]
- 更新 [[topics/topic-1]]
- 新增交叉引用 3 处

## [YYYY-MM-DD HH:MM] query | 关于X的问题
- 创建 [[synthesis/analysis-x]] 作为答案
- 更新 [[topics/topic-2]]
```

## 输出格式

根据问题性质，可输出：

- Markdown 页面（默认）
- Marp 幻灯片（`---` 分页）
- 对比表格（Markdown table）
- 图表（描述后用工具生成）
- 纯文本解释

## 注意事项

1. **原始素材不可修改**：`raw/` 目录只读，所有分析写入 `wiki/`
2. **矛盾显式标注**：发现矛盾时，在相关页面明确标注并说明
3. **引用可追溯**：每个主张追溯到来源页面
4. **适度抽象**：实体页面保持通用，来源页面保持具体
5. **及时更新**：每次摄入后立即更新所有相关页面
6. **中文优先**：内容使用中文，保留原文术语

## 扩展工具

- **搜索**：目前靠 index.md 导航，规模增长后可引入 qmd 等搜索工具
- **Obsidian**：推荐用 Obsidian 浏览 wiki，利用图视图观察知识结构
- **Web Clipper**：用 Obsidian Web Clipper 快速收集网页内容
- **Dataview**：可基于 frontmatter 生成动态列表
