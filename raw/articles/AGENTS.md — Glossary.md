**AGENTS.md** is an open standard file format for providing project-specific instructions to AI coding [agents](https://latentpatterns.com/glossary/agent). Placed in a repository’s root directory, it contains the context an agent needs to work effectively — build commands, testing procedures, coding conventions, architecture notes, and security considerations — in plain Markdown with no required fields or special syntax. Where a `README.md` is written for human contributors, an `AGENTS.md` is written for the [tool-calling](https://latentpatterns.com/glossary/tool-calling) systems that read, edit, and generate code alongside them.

The idea of a single, vendor-neutral instruction file emerged from the fragmentation of tool-specific configuration files — `.cursorrules`, `.windsurfrules`, `.clinerules`, `GEMINI.md`, and Anthropic’s [CLAUDE.md](https://latentpatterns.com/glossary/claude-md) — each readable by only one tool. Geoffrey Huntley proposed consolidating these into a single standardized filename, `AGENT.md` (singular), published as an informational RFC at [agentmd/agent.md](https://github.com/agentmd/agent.md) in July 2025. Independently, Amp had adopted `AGENT.md` in May 2025 and secured the `agent.md` domain. When OpenAI announced they would standardize on `AGENTS.md` (plural) and acquired the `agents.md` domain, Amp aligned with the plural form and redirected their domain to OpenAI’s. The format was transferred to the Agentic AI Foundation (AAIF) under the Linux Foundation in December 2025, and by early 2026 public adoption had reached tens of thousands of open-source projects.

The key design decision in AGENTS.md is its **discovery hierarchy**. Tools that support the format walk from a global scope down to the current working directory, concatenating every `AGENTS.md` they find along the way. Files closer to the working directory appear later in the prompt and effectively override earlier guidance:

The exact discovery behavior varies by tool. OpenAI Codex, for example, checks for `AGENTS.override.md` at each level (a temporary override that takes precedence over `AGENTS.md` in the same directory) and enforces a default 32 KiB concatenation limit. Other tools may implement a simpler lookup — reading only the root `AGENTS.md` — or add their own extensions. The specification intentionally leaves these details to each implementation. The shared contract among tools that support AGENTS.md is straightforward: they look for that filename and treat it as instruction context. This hierarchical loading makes the format especially well-suited for [monorepos](https://latentpatterns.com/glossary/monorepo), where the root file can describe shared conventions and each package can add its own build commands, test runners, and domain-specific guidance without modifying the root.

The defining characteristic of AGENTS.md — and the reason for its rapid adoption — is **cross-tool compatibility**. A single file is now read natively by a growing set of coding tools (including OpenAI Codex, GitHub Copilot, Cursor, Windsurf, Amp, and others), in contrast to vendor-specific instruction files like [CLAUDE.md](https://latentpatterns.com/glossary/claude-md), which are read only by a single tool. Claude Code does not natively read `AGENTS.md` (it reads only `CLAUDE.md`), but teams bridge the gap with a symlink — see below. AAIF governance helps the format evolve as a shared standard rather than fragmenting into competing, vendor-specific specifications.

For teams that already maintain a vendor-specific file, the recommended migration strategy uses **symlinks** to avoid content duplication. The canonical content lives in `AGENTS.md` (version-controlled), and a symlink at `CLAUDE.md → AGENTS.md` ensures that Claude Code — which reads [CLAUDE.md](https://latentpatterns.com/glossary/claude-md) by default — continues to find its instructions. The symlinked file can be added to `.gitignore` so only the `AGENTS.md` is tracked. Claude Code resolves symlinks transparently, reading and editing the target file directly. This approach lets a team author instructions once while maintaining backward compatibility with any tool that expects a specific filename.

The trade-off of any instruction file is [context window](https://latentpatterns.com/glossary/context-window) consumption. Every byte of `AGENTS.md` is injected into the agent’s prompt, competing with conversation history, [tool calling](https://latentpatterns.com/glossary/tool-calling) definitions, and the model’s own output for space within the token budget. Concise, specific instructions outperform verbose documentation — “run `pnpm test` before committing” is more useful than a paragraph explaining what tests are. There is also a security surface: because agents treat instruction files as trusted guidance, a malicious `AGENTS.md` in a cloned repository can act as a [prompt injection](https://latentpatterns.com/glossary/prompt-injection) vector, directing the agent to execute unintended commands. Teams should review instruction files in third-party repositories with the same scrutiny they apply to build scripts and CI configurations.
