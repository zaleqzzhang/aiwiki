# AI Wiki —— 个人 AI 知识库

基于 **LLM Wiki 方法论**构建的个人知识库。核心理念：LLM 持续维护一个结构化的 wiki，知识在摄入时编译、交叉引用、标注矛盾，形成可累积的知识资产，并直接支撑技术写作。

## 当前规模

| 层级 | 数量 | 说明 |
|------|------|------|
| 原始素材 | 16 篇 | `raw/articles/` |
| 来源摘要 | 16 篇 | 每篇原文一个摘要页 |
| 实体页面 | 17 个 | 人物、组织、概念、工具 |
| 主题页面 | 3 个 | 按领域组织的主题综述 |
| 综合分析 | 1 篇 | 跨来源的观点对比 |
| 概念卡片 | 7 张 | 面向写作的概念讲解 |
| 设计笔记 | 4 篇 | "为什么这么做"的深度分析 |
| 认知缺口 | 3 张 | 读者常见误解的精准击中 |
| 视角卡片 | 2 张 | 同一问题的多角度思辨 |

当前聚焦领域：**AI Agent 系统设计**（Harness Engineering、Agent Memory、Multi-Agent Collaboration）

## 工作流

### 四大流程

| 流程 | 触发 | 做什么 |
|------|------|--------|
| **Ingest** | 新素材放入 `raw/` | 创建来源摘要 → 更新实体/主题 → 增量更新 craft/ → 更新索引 |
| **Query** | 提问 | 读索引定位 → 深入阅读 → 综合分析（有价值的答案沉淀为新页面）|
| **Craft** | 准备写文章 | 从 concepts/ 确保概念准确 → design-notes/ 补深度 → gaps/ 选切入角度 → perspectives/ 增思辨 → drafts/ 起草 |
| **Lint** | 定期 | 检查矛盾、过时信息、孤立页面、缺失引用 |

### Craft 更新策略

创作层有三级更新机制：

- **级别一：Ingest 增量微调**（自动）—— 摄入时检查新素材是否命中现有卡片，增量更新
- **级别二：写作目标备料**（说"写一篇 X"触发）—— 全面检查素材是否够用，缺什么补什么
- **级别三：系统盘点**（说"盘一下 craft"触发）—— 全面审计重建

### 联网信息处理

- **前置规则**：每次联网前必须先询问阿正并获得同意，不得自动联网
- **辅助性信息**（补充背景、验证事实）→ 不入库，用完即弃
- **实质性信息**（新文章、新项目、新观点）→ 走完整 Ingest 流程，在 log.md 标注"来源：联网获取"
- **核心约束**：知识库每条信息必须可追溯来源

## 创作体系

面向「理解导向」的技术深度写作，目标受众是**对 AI 感兴趣的技术从业者**。

### 写作路径

```
① 选切入角度 → gaps/（读者"以为懂但搞混了"的地方）
② 搭概念骨架 → concepts/（选 2-4 个概念排成递进链）
③ 填深度内容 → design-notes/ + perspectives/
④ 起草 → drafts/
```

### 写作原则

| 原则 | 怎么做 |
|------|--------|
| 概念先行 | 先讲清"是什么"和"为什么这么设计" |
| 层层递进 | 从直觉反应 → 设计考量 → 深层启发 |
| 细节服务理解 | 每个技术细节回答"帮读者理解什么" |
| 有观点有边界 | 有判断，同时说明适用范围 |
| 填补认知缺口 | 击中"以为懂但搞混了"的地方 |

## 使用方式

推荐用 **Obsidian** 浏览，利用双链和图视图观察知识结构。

### 快捷操作

| 说法 | 效果 |
|------|------|
| 把素材丢到 `raw/` | 触发完整 Ingest 流程 |
| "写一篇关于 X 的文章" | 备料 + 起草 |
| "盘一下 craft" | 全面审计创作层 |
| "基于 [gap 名] 写一篇" | 从认知缺口切入写文章 |
